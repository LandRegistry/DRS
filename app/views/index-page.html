Digital Registration Service

Introduction to Digital Registration Service

The problem

Conveyancers applying to change the register generate a lot of requisitions, especially when using the AP1 form, which is needed for many transactions. It’s a complex, multi-purpose form wrapped in legalese guidance that’s not always understood by users. Our users are often clerical staff at conveyancers, some of whom have learned to complete the form by rote. It’s easy to make mistakes of accuracy or omission, and most lead to requisitions. This project took replacing the paper AP1 with a digital system as a starting point.

Aims of the service:

  • make it easier for users to provide exactly (and only) the data needed
  • reduce requisitions caused by inaccurate data or missing documents
  • save HMLR time and expense in handling paper forms, and issuing and handling requisitions
  • save customers time and expense in completing forms and managing requisitions
  • make it easier for customers to manage their applications
  • ongoing, clean the data in applications and on the register

Methods to achieve this:

  • digitised data capture
  • validated formats and data where possible
  • prompts and guidance around common requisition points
  • save and resume applications
  • enabling automation where possible
  • enabling integration of compatible applications where possible

Project so far

In 2020, we've designed, tested, developed and deployed discharges, transfers for value and charges, allowing users to submit some of the most common transaction combinations.

Over the Summer, we've created an accessibility statement for the service, tested and settled on an MVP for fees calculation and created an in-service feedback function for users.
