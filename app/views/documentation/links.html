Links

When to open in an external window and how to display this. The pattern is as follows:

  • We often have to open links in a new window as otherwise it would interrupt a user’s application and risk loss of data.
  • For pages like the accessibility statement, the user is not mid-journey and the statement has already opened in a new window, so allowing links to open in the same window is preferable.
  • If the content is short and contains one or two links, adding '(link opens in a new tab)' to each URL is fine. Same applies if there is one link in a denser body of text.
  • If there are several links in a longer body of text, we use the icon (e.g. Declaration statement before submission, screenshot below)
  • We can also use the icon where there isn’t room for additional text, e.g. Portal Refresh uses the icon to link to guidance from the View My Applications table, which is data heavy.
  • The alt-text for the icon should read ‘Link opens in a new tab' unless the link opens in a window.
  • The footer in Portal should have text above the links which says ‘Links open in new windows’ - the footer should be consistent with that of View My Applications.
  • The specific icon used was approved by design leadership and is used in VMA and DRS.

Basically, only use the icon as a last resort.

These guidelines are subject to each individual case.
