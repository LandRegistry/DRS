Service status errors

Service errors are potentially terminal points in a users journey, whether due to a problem with the service or user behaviour which is prohibited (e.g. bots) Below are the current templates for service errors.

Figma templates for these can be found here.

Error screen templates

User-triggered error

For errors a user has caused, for example going to a forbidden page.

User-triggered security error

For errors to stop unauthorised/fraudulant activity.

Recoverable service or API errors

For use when it's our problem...

Unrecoverable service or API errors

For use when it's our problem and we can't direct the user to the previous page. This isn't great from a user experience point of view, so should only be used when absolutely necessary.
